# Schema Design and Constraints

## Summary

This chapter covers classes, properties, cardinality, and constraints needed
to design a rigorous schema before populating a knowledge graph with real
data. It works through the one-to-one, one-to-many, and many-to-many
relationship types, domain-and-range declarations, and inheritance, then
ties them together into a repeatable schema-design process. Readers will
finish able to design a defensible schema for an original domain rather than
improvising structure as data arrives.
